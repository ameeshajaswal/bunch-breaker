## Transit Data Challenge 2026

### Problem Statement

On TTC bus routes, where does vehicle bunching cause the greatest mid-route headway variance during peak hours, and which specific intersections should a transit supervisor prioritize to restore even service?

### Solution

The Bunch Breaker is an optimization tool that recommends exactly where to deploy limited supervisors to fix the most bus bunching hotspots across multiple routes.

### Features

- 🗺️ **Interactive Map** - Leaflet map with color-coded hotspots (🔴 Critical, 🟠 High, 🟡 Moderate)
- ⚖️ **3 Optimization Strategies** - Greedy (max severity), Weighted (severity × riders), Equity (low-income priority)
- 📊 **Real-time KPI Cards** - Coverage %, Riders helped, Avg reduction, Equity score
- 📄 **CSV/PDF Export** - Download supervisor deployment reports
- 🔗 **Shareable URLs** - Parameters sync to URL for bookmarking
- ♿ **High Contrast Mode** - Accessibility compliant
- 🔒 **Privacy First** - 100% open/synthetic data, zero PII

### Setup Instructions

#### Prerequisites

- Python 3.12+
- Node.js 18+
- Git

#### Backend Setup

```bash
# Clone repository
git clone https://github.com/ameeshajaswal/transit-data-challenge-2026.git
cd transit-data-challenge-2026

# Create virtual environment
python -m venv venv
venv\Scripts\activate  # On Windows
# source venv/bin/activate  # On Mac/Linux

# Install dependencies
pip install -r requirements.txt

# Download TTC GTFS data
python backend/src/data/download_gtfs.py

# Generate synthetic ridership
python backend/src/data/create_synthetic_ridership.py

# Compute headway variance
python backend/src/data/compute_headways.py

# Create hotspot database
python backend/src/data/create_hotspot_table.py

# (Optional) Add visible hotspots for demo
python create_fresh_db.py

# Run backend server
uvicorn backend.src.api.main:app --reload --port 8000
Frontend Setup (New Terminal)
bash
cd frontend
npm install
npm run dev
Open Browser
Frontend: http://localhost:5173

Backend API: http://localhost:8000/api/health
